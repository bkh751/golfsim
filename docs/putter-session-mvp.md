# Putter Session MVP

## 1. 목표
- 휴대폰 센서를 이용해 퍼팅 스트로크의 거리감과 시작 라인을 연습한다.
- 실제 골퍼가 짧은 퍼트부터 롱 퍼트까지 같은 기준으로 반복 훈련할 수 있게 한다.
- 센서가 없는 환경에서도 같은 물리 모델로 fallback QA를 수행한다.

## 2. 이번 이터레이션 범위
### PM
- flat green 하나와 컵 하나만 다룬다.
- 목표는 `거리 조절`, `직진성`, `템포 피드백` 3가지다.
- 현재 레인지 세션을 흔들지 않고 별도 퍼터 세션 페이지로 제공한다.

### 기획
- 사용자 플로우:
  1. 퍼터 세션 진입
  2. 목표 거리와 그린 속도 선택
  3. 센서 연결 또는 fallback 선택
  4. 캘리브레이션
  5. 스트로크 대기
  6. 퍼트 결과 재생
  7. 거리 오차, 라인 오차, 템포 확인
  8. 같은 거리 또는 다른 거리로 반복
- 상태 전이:
  - `ready -> calibrating -> armed -> tracking -> result -> ready`
- 입력:
  - 목표 거리
  - 그린 속도
  - DeviceMotion 기반 스트로크
  - 수동 fallback 스트로크 강도
- 출력:
  - 볼 스피드
  - 거리 오차
  - 라인 오차
  - 템포 / 품질 점수
  - 최근 퍼팅 이력

### 디자인
- 모바일 우선 단일 페이지로 설계한다.
- 상단에는 `현재 상태`, `타깃`, `직전 결과`, `센서 상태`를 요약 카드로 둔다.
- 중앙에는 탑다운 퍼팅 그린을 두고 컵, 기준선, 최근 결과를 같은 좌표계로 겹쳐 보여준다.
- 우측 또는 하단 패널에는 센서 버튼, fallback, 결과 지표, 최근 이력을 둔다.
- 정보는 많지 않게 유지하고, 직전 한 퍼트의 결과가 가장 먼저 읽히게 한다.

### 개발
- 새 모듈 `putt-model.js`에서 퍼트 물리와 스트로크 분석을 분리한다.
- `putter-session.html`에서 센서 입력, fallback, 캔버스 렌더, 결과 UI를 구현한다.
- 테스트는 모델 단위와 브라우저 페이지 단위 둘 다 추가한다.

### Gameplay QA
- 실제 센서 경로와 fallback 경로가 같은 물리 결과를 쓰는지 본다.
- 권한 거부, 센서 미지원, 캘리브레이션 실패, 약한 스트로크를 모두 확인한다.
- 최근 퍼팅 이력과 거리 오차가 반복 훈련에 실제로 도움이 되는지 확인한다.

## 3. Non-goal
- 3D 퍼팅 그린 지형
- 경사, 브레이크, 잔디 결
- 클럽 모델링과 퍼터 페이스 충돌 정밀 계산
- 멀티홀 또는 코스 플레이
- 엔진 포팅

## 4. 수용 기준
1. 사용자는 퍼터 세션에 진입해 목표 거리와 그린 속도를 바꿀 수 있다.
2. 센서를 지원하는 기기에서는 권한 요청, 캘리브레이션, 스트로크 대기 흐름이 동작한다.
3. 실제 스트로크 또는 fallback 입력으로 볼이 굴러가고 결과가 재생된다.
4. 결과 화면에는 거리 오차, 라인 오차, 볼 스피드, 템포/품질이 표시된다.
5. 최근 퍼팅 이력은 최소 6개까지 유지되고, 반복 훈련에 필요한 비교가 가능하다.
6. 센서 권한 거부나 미지원 상태에서도 fallback 경로로 같은 물리 모델을 검증할 수 있다.
7. 현재 레인지 세션은 회귀 없이 그대로 유지된다.
