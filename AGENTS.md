# AGENTS.md
- `golfsim`의 기본 가치는 일관된 UX, 신뢰 가능한 결과, 작은 범위의 안전한 변경이다.
- 우선순위는 `사용자 지시 > 현재 Issue > docs/product.md > AGENTS.md > 기존 관례`이며, 충돌 시 추정하지 않는다.
- 범위, 수용 기준, 결정 변경은 항상 GitHub Issue 본문에 반영하고 코멘트만으로 바꾸지 않는다.
- `planning/design/dev/gameplay_qa/review`는 자기 역할의 판단만 맡고, 넘는 결정은 해당 역할이나 PM으로 다시 넘긴다.
- 작업은 필요한 파일만 읽고 작게 나누어 진행하며, 구현에는 테스트 결과와 남은 리스크를 같이 남긴다.
- 세션 컨텍스트 사용량은 항상 `15% 미만`을 유지하고, `15% 이상`이 되면 즉시 컨텍스트를 클리어한 뒤 짧은 상태 요약만 남기고 계속한다. 이 규칙은 하위 에이전트에도 동일하게 적용한다.
- 커밋은 응집된 변경 단위와 한글 메시지를 기본으로 하고 가능하면 Issue를 참조하며, `amend/reset/force push`는 요청 시만 쓴다.
- Harness는 `hooks advisory`, `.codex/orchestrator` 상태 저장, `.codex/harness` 로그 저장, `dashboard read-only` 원칙을 따른다.
- 메인 대화는 자유 형식으로 하고, Harness 내부 협업은 routed p2p `steered_mesh`를 쓰며 `compact/kv/symbolic` 패킷과 `interrupt` 우선 흐름을 허용하고 고정 템플릿은 금지한다.
- 완료 기준은 승인 범위와 수용 기준 충족, 테스트 통과 또는 누락 사유 기록, 미해결 의사결정을 숨기지 않는 것이다.
